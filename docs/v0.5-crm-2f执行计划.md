## 2. Step 2f-1：保存视图增强（GAP-06）

**目标：** 用户保存列表视图时可设置 **公开 / 默认 / 钉选**；API 已有字段，补齐 Web UI。

**目标文件：**

- `apps/web/src/views/crm/Leads.vue`
- `apps/web/src/views/crm/Customers.vue`
- `apps/web/src/api/client.js`（确认 `createView` / `updateView` 传参）

**实施步骤：**

1. 在「保存视图」对话框增加：
   - `el-checkbox`：**设为默认视图** → `is_default`
   - `el-checkbox`：**钉选到侧栏** → `is_pinned`（需 `crm.view.save_own`）
   - `el-checkbox`（仅 admin 或 `crm.view.manage_public`）：**公开给全公司** → `is_public`
2. `submitSaveView()` 调用 `crmApi.createView({ ..., is_default, is_pinned, is_public })`。
3. 若用户勾选默认，保存后 `activeViewId` 切到新视图并刷新列表。
4. 编辑已有视图（可选 P1）：列表视图下拉旁增加「管理视图」入口，可 PATCH 上述字段。

**严格验收：**

| 编号 | 通过标准 |
|------|----------|
| V2f-1-1 | `POST /crm/views` body 含 `is_pinned` / `is_default` / `is_public` 时返回 201 且字段持久化 |
| V2f-1-2 | `Leads.vue` 保存对话框含 `is_pinned` 或文案「钉选」 |
| V2f-1-3 | `Leads.vue` 含 `is_default` 或「默认视图」 |
| V2f-1-4 | `Customers.vue` 同上（保存视图增强） |
| V2f-1-5 | 非 admin 用户 `is_public=true` 且无 `crm.view.manage_public` → 403 |

## 4. Step 2f-3：导入 `on_duplicate=update`（GAP-02）

**目标：** 重复键第二次导入时 **更新** 当前用户可见记录，而非仅 skip。

**目标文件：**

- `apps/api/app/services/crm/import_service.py`
- `apps/web/src/components/crm/CrmImportDialog.vue`（选项 UI：skip / update）

**实施步骤：**

1. `run_import()` 中当 `on_duplicate=update` 且命中 `duplicate_key`：
   - 查找租户内同键记录；若不可见（scope）→ 该行 **error** 或 **skip**（与 BR-IMPORT-02 一致）。
   - 若可见 → PATCH 更新映射字段，`success_count++`。
2. 预览阶段对 update 模式标注「将更新」而非「将跳过」。
3. Web 导入对话框「重复处理」下拉：`跳过` / `更新`。
4. 补充单元或验收：同手机号两次导入，第二次 `company_name` 变化。

**严格验收：**

| 编号 | 通过标准 |
|------|----------|
| V2f-3-1 | 首次导入 mobile=M → `success_count=1` |
| V2f-3-2 | 第二次同 M、`on_duplicate=update`、改公司名 → `success_count=1` 且 DB 公司名已更新 |
| V2f-3-3 | 第二次 `on_duplicate=skip` 仍 → `skip_count=1`（回归） |
| V2f-3-4 | sales B 不可见 sales A 的线索时，B 导入 A 的手机 update → 该行 error/skip，不越权 |
| V2f-3-5 | `CrmImportDialog.vue` 含 `update` 或 `on_duplicate` 选项 UI |
