## 0. 执行规则（必须遵守）

### 0.1 一步一验

1. **只允许顺序执行** Step 清单；不得跳过。
2. 每个 Step 完成后必须：
   - 跑该 Step 的 **专项验收**（下表「验收命令」）→ 全部 PASS
   - 跑 **回归门禁**（§0.3）→ PASS
3. 任一 FAIL → **停止**，修复后从该 Step 重跑，不得进入下一步。

### 0.2 Step 状态标记

| 标记 | 含义 |
|------|------|
| ⬜ | 未开始 |
| 🔄 | 进行中 |
| ✅ | 本 Step 专项 + 回归均 PASS |

### 0.3 每步回归门禁（固定）

```bash
cd apps/api
python tests/run_crm_all.py --through 2f
```

| 编号 | 通过标准 |
|------|----------|
| RG-CRM | `run_crm_all.py --through 2f` 退出码 0 |
| RG-AGENT | Phase E 起额外跑 `python tests/run_agent_a_c.py` |

> Phase A（仅文档）回归：仅 RG-CRM；`verify_advisor_0a.py` 专项 PASS 即可。

### 0.4 Phase 结束门禁

```bash
python tests/run_advisor_all.py --through <阶段名>
```

示例：`--through 0a` 只跑到 `verify_advisor_0a.py`。
