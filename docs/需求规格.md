## 1. 引言

### 1.1 目的

本文档定义「AI 内容获客系统」的功能需求、非功能需求、接口约束与验收标准，供产品、开发、测试使用。

### 1.2 产品愿景

为中小财务服务公司提供 **AI 辅助营销内容生产 → 内容管理 → 公众号发布（Mock/真实）→ 其他平台导出** 的 SaaS 能力。支持 **一人加入多家公司、企业内自定义角色与功能权限、平台默认 AI 额度 + 租户自有 Key**，降低获客内容的人力成本，并预留多行业与知识库扩展。

### 1.3 术语

| 术语 | 定义 |
|------|------|
| 租户 / 公司（Tenant） | 一家企业，数据隔离单元 |
| 账号（User） | 全局登录身份（手机号 + 密码），与具体公司解耦 |
| 成员关系（Membership） | 某账号在某公司的成员身份，含角色与启用状态 |
| 企业角色（Tenant Role） | 公司内角色（内置 admin/editor 或自定义），绑定功能权限 |
| 当前公司（Active Tenant） | 用户登录后选定的工作空间，决定可见数据与权限 |
| 功能权限（Permission） | 可勾选的原子能力，如 `content.create`、`dashboard.view_all` |
| 行业包（Industry Pack） | 平台预置的行业模板、公共知识、合规规则集合 |
| 内容（Content） | 一次 AI 生成或编辑后的营销稿件 |
| Mock 发布 | 开发期模拟微信公众号发布，不调用微信 API |
| RAG | 检索增强生成，从知识库取片段注入 Prompt |
| LLM Provider | 大模型供应商标识，如 deepseek、openai_compatible、dashscope |
| platform_admin | 平台运营账号，管理全站，不参与普通企业权限体系 |
| 测试验证码 | 开发/测试环境短信 Mock 固定码 **1111**（`SMS_MOCK_CODE`） |

**UI 参考：** 企知道「科创空间」Web 与 APP/H5 的信息架构与视觉气质（见 §3.11）。

**客户端范围（强制）：** 凡本文 **v0.3 起新增或修订** 的功能（含企业与权限、忘记密码、平台额度、选/切换公司等），均须 **Web 与 H5 同步实现、同步验收**，除非表中单独标注「仅 Web」。

### 1.4 范围

**当前版本包含：**

- Web 管理端（Vue 3）、移动端 uni-app（H5）
- 后端 API（FastAPI）、数据库（PostgreSQL / 开发期 SQLite）
- 可插拔 LLM；**平台默认 Key + 租户免费额度**（正文成功扣 1 次，方案不扣）
- 租户/平台知识库、品牌与个人偏好
- 企业 **角色与成员管理**、**一人多公司切换**
- 平台 **管理后台**（`/admin`，仅 Web）：租户/账号运营、转移管理员等

**暂不包含：**

- 小红书/抖音官方自动发布 API
- 计费与支付、Frappe CRM 深度集成
- 向量 RAG（P2，MVP 关键词检索）
- 微信 wx.login（P2）
- **微信小程序原生包**（当前 H5 为准；与 Web 能力对齐）

| ID | 需求 | 优先级 | 验收标准 |

- 平台额度展示与创作时 AI 来源（platform / tenant）

## 2. 用户与角色

### 2.1 用户画像

**小财务公司市场/运营人员**：非技术背景，需模板化创作与团队协作；内容涉及财税合规。

### 2.2 账号与公司

- 一个 **账号** 可拥有 **0～N 条成员关系**（加入多家公司）
- 每条成员关系：**一个公司 + 一个企业角色 + 启用/禁用**
- 登录后若多家公司，**必须先选择当前公司**；使用期间可 **切换公司**
- 切换公司后：数据、菜单、权限随 **当前公司** 变化；**个人偏好全局共用**（见 FR-BRAND-10）

### 2.3 企业内置角色

| 角色 code | 名称 | 说明 |
|-----------|------|------|
| `admin` | 企业管理员 | 系统内置，不可删除；默认拥有全部企业内权限；至少保留 1 名 admin |
| `editor` | 编辑 | 系统内置，不可删除；**新加入成员默认角色**；最小权限集（见 §2.4） |

### 2.4 自定义角色

- 企业 admin 或拥有 `team.role.manage` 的角色可 **新建/编辑/删除** 自定义角色
- 通过 **功能权限勾选** 配置能力（见 §2.5）
- 内置 `admin`、`editor` 不可删除；`admin` 权限集固定全开（不可削弱至无法管理）

### 2.5 功能权限清单（Permission Catalog）

权限以字符串 **code** 存储；在 **设置 → 角色与成员 → 角色 → 权限勾选** 中配置（含自定义角色）。

#### 创作与内容

| 权限 code | 说明 | editor 默认 |
|-----------|------|-------------|
| `content.create` | 营销创作（方案 + 生成正文） | ✅ |
| `content.list_own` | 内容库列表，**仅本人**作为作者 | ✅ |
| `content.view_own` | 查看**本人**文章详情 | ✅ |
| `content.list_all` | 内容库，**全公司** | ❌ |
| `content.view_all` | 查看**任意**文章 | ❌ |
| `content.edit` | 编辑内容 | ❌ |
| `content.delete` | 删除内容 | ❌ |
| `content.export` | 导出小红书/抖音等 | ❌ |
| `content.schedule` | 发布日历 / 排期 | ❌ |
| `content.publish` | 发布到公众号 | ❌ |

#### 知识库

| 权限 code | 说明 | editor 默认 |
|-----------|------|-------------|
| `knowledge.view` | 查看企业知识库 | ❌ |
| `knowledge.manage` | 上传/删除企业资料 | ❌ |

#### 设置

| 权限 code | 说明 | editor 默认 |
|-----------|------|-------------|
| `preference.manage` | **我的偏好**（全局，见 FR-BRAND-10） | ✅ |
| `brand.manage` | 品牌设置 | ❌ |
| `wechat.manage` | 公众号绑定 | ❌ |
| `llm.manage` | 租户 AI Key 配置 | ❌ |
| `tenant.manage` | **企业基本信息**（名称、信用代码、行业等） | ❌ |

#### 工作台与数据看板

| 权限 code | 说明 | editor 默认 |
|-----------|------|-------------|
| `dashboard.view` | 访问工作台 | ✅ |
| `dashboard.view_all` | 工作台 **全公司** 统计 | ❌ |
| `analytics.view` | 访问数据看板 | ✅ |
| `analytics.view_all` | 数据看板 **全公司** 统计 | ❌ |

**统计范围规则（FR-DASH / FR-ANLY）：**

- 有 `*.view_all` → 按 **当前公司 tenant** 全量聚合
- 仅有 `*.view`、无 `*.view_all` → 强制 **`author_id = 当前用户`**（本人数据）
- API **服务端过滤**，不可仅靠前端隐藏

#### 团队

| 权限 code | 说明 | editor 默认 |
|-----------|------|-------------|
| `team.member.view` | 查看成员列表 | ❌ |
| `team.member.manage` | 添加成员、改角色、禁用、重置密码 | ❌ |
| `team.role.manage` | 新建/编辑/删除自定义角色及权限 | ❌ |

**成员管理约束：**

- 拥有 `team.member.manage` 的自定义角色 **可以** 管理普通成员
- **非 admin 操作者** 不得：新增 admin、删除 admin、修改 admin 的角色或禁用 admin
- 系统至少保留 **1 名 admin**

**权限 UI 分组示例：** 工作台（view / view_all）→ 数据看板 → 内容 → 知识库 → 设置 → 团队。不勾 `view` 时对应 `view_all` 灰显不可选。

### 2.6 平台管理员与后台边界

| 角色 / 入口 | 说明 |
|-------------|------|
| **platform_admin** | 平台运营；Web **`/admin`**；**不参与**企业 Membership 权限体系 |
| **企业 admin** | 租户内管理员；在用户端 **设置 → 企业信息 / 角色与成员**（Web+H5） |

**职责划分：**

| 能力 | 平台 `/admin` | 用户端设置（企业 admin） |
|------|---------------|-------------------------|
| 公共知识库、AI 助手、平台 AI | ✅ | ❌ |
| 全站内容巡查 | ✅ | ❌ |
| **租户列表、查看企业成员、转移管理员** | ✅ | ❌ |
| **全局账号**（跨公司 Membership 汇总） | ✅ 账号管理 | ❌ |
| 企业信息、信用代码 | 只读（租户详情） | ✅ 编辑（tenant.manage） |
| 企业内角色/成员/权限勾选 | ❌ | ✅ FR-TEAM |
| 忘记密码 | ❌（公共登录页 Web+H5） | 用户自助 |

**客户端：** 平台管理后台 **仅 Web**；不要求 H5 版 `/admin`。

## 3. 功能需求

### 3.1 账户、公司与成员（FR-AUTH / FR-TENANT / FR-TEAM）

| ID | 需求 | 优先级 | 验收标准 |
|----|------|--------|----------|
| FR-AUTH-01 | 创建公司时选择主行业 | P0 | Tenant 绑定 industry_code |
| FR-AUTH-02 | 手机号 + 密码登录，返回 JWT | P0 | Web/H5 登录成功 |
| FR-AUTH-04 | 租户数据严格隔离 | P0 | 当前公司外数据不可访问 |
| FR-AUTH-05 | 微信一键登录 | P2 | 部署后小程序 |
| FR-AUTH-10 | 账号与成员关系分离 | P0 | 同一手机号一条 User；多公司通过 Membership |
| FR-AUTH-11 | 注册：创建公司 | P0 | Tenant + 内置 admin/editor + 首成员 admin |
| FR-AUTH-12 | 加入公司 | P0 | 添加 Membership，默认角色 **editor** |
| FR-AUTH-13 | 登录后选择当前公司 | P0 | Membership≥2 时 **必须先选一家** |
| FR-AUTH-14 | 切换当前公司 | P0 | token 含 active_tenant_id；数据/权限随切换更新 |
| FR-AUTH-16 | 从公司移除成员 | P1 | 禁用 Membership，不删全局账号 |
| FR-AUTH-17 | 解散公司 | P2 | 仅 admin，二次确认 |
| FR-AUTH-18 | **忘记密码入口** | P0 | 登录页提供「忘记密码」链接（Web/H5） |
| FR-AUTH-19 | **手机验证码找回密码** | P0 | 输入已注册手机号 → 收验证码 → 设置新密码 → 提示成功并跳转登录 |
| FR-AUTH-20 | 找回密码-发送验证码 | P0 | 已注册且启用手机号；60s 频控；**Mock 固定码 1111**（Web/H5 一致） |
| FR-AUTH-21 | 找回密码-校验与重置 | P0 | 验证码正确且未过期后允许提交新密码（≥8 位）；成功后旧密码失效 |
| FR-AUTH-22 | 与验证码登录区分 | P0 | `sms/login` 用于**登录**；找回密码**仅改密不自动签发 token**（除非产品另行约定自动登录） |

**忘记密码流程（FR-AUTH-18～21）：**

```
登录页 → 忘记密码
  → 输入手机号 → 获取验证码
  → 输入验证码 + 新密码 + 确认密码
  → 提交成功 → 返回登录页用新密码登录
  → （若多公司）再按 FR-AUTH-13 选择公司
```

**接口概要（公开，无需登录）：**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/auth/password/forgot/send-code` | 发送找回密码验证码（body: phone） |
| POST | `/api/v1/auth/password/forgot/reset` | 验证码 + 新密码重置（body: phone, code, password） |

**与现有能力关系：**

| 能力 | 场景 | 是否登录 |
|------|------|----------|
| 密码登录 FR-AUTH-02 | 日常登录 | 否 → 是（JWT） |
| 验证码登录 | 免密登录 | 否 → 是 |
| **忘记密码** | 用户自助改密 | 否 → 否（改完后自行登录） |
| 企业 admin 重置成员 FR-TEAM-09 | 管理员帮同事改密 | 是（需 team.member.manage） |
| 平台 admin 重置 | 运营后台 | 是（platform_admin） |

**安全与体验（NFR，见 §5）：**

- 未注册手机号：发送验证码返回明确提示（不泄露是否存在，或统一文案「若已注册将发送验证码」— 实现时二选一并在测试说明）  
- 验证码有效期、发送间隔与登录验证码 **可共用配置**（`SMS_CODE_EXPIRE_SEC`、`SMS_SEND_INTERVAL_SEC`）  
- 建议验证码 **场景隔离**（login / reset_password），避免登录验证码被用于改密  
- 重置成功后 **使该账号既有 JWT 失效**（可选 P1：改密后清 token）  
- **测试环境：** `SMS_PROVIDER=mock`，`SMS_MOCK_CODE=1111`；登录与找回密码均使用该码；Web/H5 测试文档统一写明 **1111**

## 4. 数据与业务规则

### 4.1 内容状态机（v0.3 简化）

```
draft → scheduled → publishing → published
draft → publishing → published（具备 publish 权限）
publishing → failed → draft 或 retry
draft → exported（导出路径，按权限）
```

**BR-01：** AI 生成默认 draft，不自动发布。

**BR-02（修订）：** 不强制审核状态机；发布权限由 `content.publish` 控制。

**BR-03：** 所有业务查询使用 **active_tenant_id** 过滤；切换公司后隔离生效。

**BR-04～07：** 同 v0.2（industry_code、合规免责声明等）。

### 4.2 逻辑数据模型（附录摘要）

```
User 1──* TenantMembership *──1 Tenant
TenantMembership *──1 TenantRole
TenantRole 1──* TenantRolePermission (permission code)
Tenant 1──* Content, KnowledgeDocument, LLMConfig, ...
User 1──0..1 UserPromptProfile（全局）
Content.tenant_id + Content.author_id
PlatformLLMConfig, TenantLLMUsage（平台额度按 tenant）
platform_admin User（独立，不走 Membership）
```

## 6. 测试与验收

### 6.1 UAT 场景（企业与权限）

1. 张三创建「甲公司」→ admin；李四加入 → 默认 editor → 仅创作 + 偏好 + **本人**内容库 + **本人**工作台/看板  
2. 李四加入「乙公司」→ 登录 **先选公司**；甲公司文章在乙公司 **不可见**  
3. 自定义角色「运营」：勾 member.manage + dashboard.view_all，**不能**改 admin 成员  
4. admin 改公司名、统一社会信用代码 → 顶栏更新；editor 无企业信息入口  
5. 李四改「我的偏好」→ 切换公司后偏好 **相同**  
6. admin 工作台/看板数字 **≥** editor 本人统计  
7. **忘记密码（Web + H5）：** 已注册手机号 → 验证码 **1111** → 新密码 → 双端旧密码失败、新密码登录成功  
8. 未注册手机号请求验证码 → 按产品约定提示；连续错误验证码拒绝重置  
9. **双端一致性：** §1.5 清单中任一场景 Web 通过后，H5 须同样通过（选公司、切换、editor 本人看板、忘记密码等）

### 6.2 UAT 场景（平台管理后台，仅 Web）

10. 平台 admin → **企业管理** → 搜公司 → 成员列表含手机号、姓名、角色  
11. 对成员李四执行 **转移管理员** → 李四为 admin、原 admin 张三为 editor → 张三用户端无「企业信息/角色与成员」编辑入口  
12. **账号管理** → 某手机号展示加入的多家公司及角色；删除账号 **不删除** 企业 Tenant 数据
