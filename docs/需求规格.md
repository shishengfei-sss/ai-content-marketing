## 1. 引言

### 1.1 目的

本文档定义「AI 内容获客系统」MVP 的功能需求、非功能需求、接口约束与验收标准，供产品、开发、测试使用。

### 1.2 产品愿景

为中小财务服务公司提供 **AI 辅助营销内容生产 → 人工审核 → 公众号自动发布 → 其他平台导出** 的 SaaS 能力，降低获客内容的人力成本，并预留多行业、租户/个人知识库与风格扩展。

### 1.3 术语

| 术语 | 定义 |
|------|------|
| 租户（Tenant） | 一家财务公司，数据隔离单元 |
| 行业包（Industry Pack） | 平台预置的行业模板、公共知识、合规规则集合 |
| 内容（Content） | 一次 AI 生成或编辑后的营销稿件 |
| Mock 发布 | 开发期模拟微信公众号发布，不调用微信 API |
| RAG | 检索增强生成，从知识库取片段注入 Prompt |
| LLM Provider | 大模型供应商标识，如 deepseek、openai_compatible、dashscope |
| LLM Adapter | 统一 chat 接口的实现，屏蔽各厂商 API 差异 |

**UI 参考（风格参照，不复制商标与素材）：** 企知道科技有限公司「科创空间」Web 管理端与 APP/H5 的信息架构与视觉气质（见 §3.10）。

### 1.4 范围

**MVP 包含：**

- Web 管理端（Vue 3）
- 移动端（uni-app，开发期 H5；部署后微信小程序）
- 后端 API（FastAPI）、任务队列（Celery）、数据库（PostgreSQL）
- 可插拔 **大模型（LLM）** 内容生成，默认 DeepSeek，可配置其他厂商
- 租户/平台知识库（finance）
- 审核工作流
- 微信公众号自动发布（部署后真实 API；开发期 Mock）
- 小红书 / 抖音 **生成 + 导出**（不自动发布）

**MVP 不包含：**

- 小红书 / 抖音官方自动发布 API
- 微信服务号 / 小程序 / 备案（推迟到部署阶段申请）
- 计费与支付
- Frappe CRM 深度集成（二期）
- 多行业 UI（架构预留，MVP 仅 finance）

## 2. 用户与角色

### 2.1 用户画像（MVP 首要）

**小财务公司市场/运营人员**

- 需持续产出公众号、小红书、抖音素材
- 非技术背景，需要模板化与一键生成
- 内容涉及财税，需审核后发布

### 2.2 系统角色

| 角色 | 权限摘要 |
|------|----------|
| **tenant_admin** | 租户内用户管理、品牌设置、公众号绑定、全部内容操作 |
| **editor** | 创作、编辑、提交审核、导出 |
| **reviewer** | 审核通过/驳回、查看内容 |
| **platform_admin** | 平台运营（二期），管理行业包与公共知识库 |

MVP 可实现 admin + editor + reviewer 三种租户内角色。

## 3. 功能需求

### 3.1 账户与租户（FR-AUTH）

| ID | 需求 | 优先级 | 验收标准 |
|----|------|--------|----------|
| FR-AUTH-01 | 租户注册时选择主行业（MVP 仅 finance） | P0 | 注册后 tenant 绑定 industry_code |
| FR-AUTH-02 | 邮箱/手机号 + 密码登录，返回 JWT | P0 | Web/H5 登录成功，token 有效 |
| FR-AUTH-03 | 租户内多用户，角色分配 | P1 | reviewer 不能改租户品牌设置 |
| FR-AUTH-04 | 租户数据严格隔离 | P0 | 租户 A 无法访问租户 B 任何 API 数据 |
| FR-AUTH-05 | 微信一键登录 | P2 | 二期 / 部署后小程序 |

### 3.2 知识库（FR-KB）

| ID | 需求 | 优先级 | 验收标准 |
|----|------|--------|----------|
| FR-KB-01 | 租户上传 PDF/TXT/粘贴文本至私有知识库 | P0 | 文件解析成功，列表可查看状态 |
| FR-KB-02 | 知识库按 industry_code 隔离 | P0 | finance 文档不参与其他行业检索 |
| FR-KB-03 | 平台预置 finance 公共知识库 | P0 | 无租户文档时生成仍可用行业 FAQ |
| FR-KB-04 | 生成时 RAG：租户 KB 优先于行业 KB | P0 | 价目表上传后生成文含正确服务名/价格 |
| FR-KB-05 | 文档删除与重新索引 | P1 | 删除后不再被检索 |
| FR-KB-06 | 向量检索（pgvector） | P2 | MVP 可用关键词检索替代 |

**扩展预留（非 MVP 必做）：**

- scope=platform | tenant | user 三级知识库
- 知识文档版本与 valid_until（政策过期）

### 3.3 品牌与个人偏好（FR-BRAND）

| ID | 需求 | 优先级 | 验收标准 |
|----|------|--------|----------|
| FR-BRAND-01 | 租户品牌：名称、人设、语气、CTA、范文 few-shot | P0 | 生成结果体现品牌名 |
| FR-BRAND-02 | 用户个人风格偏好（语气微调） | P2 | MVP 可简化 |
| FR-BRAND-03 | 用户全局个人提示词（textarea） | P1 | 勾选 apply 后生成口吻变化 |
| FR-BRAND-04 | 租户禁止成员自定义提示词 | P2 | 二期开关 |

### 3.4 AI 内容生成（FR-GEN）

| ID | 需求 | 优先级 | 验收标准 |
|----|------|--------|----------|
| FR-GEN-01 | **可插拔 LLM**：统一 `LLMProvider` 接口，默认 DeepSeek | P0 | 配置 DeepSeek KEY 后可生成 |
| FR-GEN-01a | 租户/平台可配置其他模型（OpenAI 兼容、通义、文心等） | P0 | 设置页切换 provider+model 后生成成功 |
| FR-GEN-01b | 配置项：provider、base_url、api_key、model、timeout | P0 | 密钥加密存储，界面脱敏展示 |
| FR-GEN-01c | 生成任务记录实际使用的 provider/model | P1 | 内容详情可查看模型来源 |
| FR-GEN-02 | 支持平台：wechat / xhs / douyin | P0 | 三种平台输出结构不同且符合模板 |
| FR-GEN-03 | 支持场景模板 ≥15 个（finance） | P0 | 报税提醒、记账介绍等可选 |
| FR-GEN-04 | Prompt 组装顺序固定（合规不可覆盖） | P0 | 每篇含免责声明 |
| FR-GEN-05 | 单次临时补充指令（不持久化） | P1 | ephemeral_instruction 仅当次生效 |
| FR-GEN-06 | 生成结果存 contents 表，status=draft | P0 | 可在内容库编辑 |
| FR-GEN-07 | 敏感词/禁用词过滤 | P1 | 含「保证不查」等拒绝或替换 |

**Prompt 组装顺序（强制）：**

1. System：合规 + JSON schema  
2. 行业模板（industry + platform + scene）  
3. RAG 检索结果  
4. tenant_brand_profiles  
5. user_style + user_prompt（可选）  
6. topic + ephemeral_instruction  

**LLM 适配架构（强制）：**

```
LLMProvider (接口: chat / stream)
├── DeepSeekProvider      ← 默认实现
├── OpenAICompatibleProvider  ← 覆盖 OpenAI / 多数国产 OpenAI 兼容网关
├── DashScopeProvider     ← 通义（可选）
└── ... 按配置扩展
```

- 环境变量默认：`LLM_PROVIDER=deepseek`  
- 租户级 `llm_configs` 表可覆盖默认（MVP 至少支持平台默认 + 租户可选一套）  
- 业务层 **禁止** 直接调用 DeepSeek SDK，只经 `LLMService.chat()`

### 3.5 内容编辑与审核（FR-REVIEW）

| ID | 需求 | 优先级 | 验收标准 |
|----|------|--------|----------|
| FR-REVIEW-01 | 内容状态机（见 4.1） | P0 | 非法状态转换返回 400 |
| FR-REVIEW-02 | 提交审核 pending_review | P0 | editor 可提交 |
| FR-REVIEW-03 | reviewer/admin 通过或驳回 | P0 | 驳回需可选原因 |
| FR-REVIEW-04 | **仅 approved 可排期发布** | P0 | draft 不能直接 publish |
| FR-REVIEW-05 | 审核记录可追溯 | P1 | content_reviews 含操作人、时间 |

| FR-WX-05 | 绑定微信服务号（OAuth 或 appid/secret） | P0 | **仅部署后 L4 验收** |

## 4. 数据与业务规则

### 4.1 内容状态机

```
draft → pending_review → approved → scheduled → publishing → published
                      ↘ draft (reject)
publishing → failed → approved (retry)
draft → exported (xhs/douyin 导出路径，可与审核流并行或导出前需 approved，MVP 建议：导出不要求审核)
```

**业务规则 BR-01：** AI 生成内容默认 draft，不得自动进入 pending_review 或 publish。

**业务规则 BR-02：** 财税类内容发布前必须经人工 approve。

**业务规则 BR-03：** 租户内所有查询必须带 tenant_id 过滤。

### 4.2 行业扩展规则

**BR-04：** 所有内容、知识库、模板表含 industry_code。

**BR-05：** 新增行业仅增 Industry Pack，不改发布主流程。

### 4.3 合规

**BR-06：** 每篇对外内容含固定免责声明（可在 tenant 层微调措辞，不可删除）。

**BR-07：** 生成内容不得构成具体税务筹划承诺或保证性表述。

## 5. 非功能需求（NFR）

| ID | 类别 | 需求 |
|----|------|------|
| NFR-01 | 性能 | 单次生成 API 响应 < 60s（含 LLM）；列表页 < 2s |
| NFR-02 | 可用性 | Web 核心流程 3 步内完成一次生成（选模板→填题→生成） |
| NFR-03 | 安全 | JWT 过期、密码 bcrypt、API 租户隔离 |
| NFR-04 | 安全 | 用户 Prompt 防注入：System 层不可被覆盖 |
| NFR-05 | 可维护 | Mock/Real 发布器同一接口 |
| NFR-06 | 部署 | Docker Compose 一键起 API+Worker+PG+Redis |
| NFR-07 | 存储 | 开发期本地 storage/；生产 OSS |
| NFR-08 | 日志 | 发布任务、LLM 调用失败可追踪 |
| NFR-09 | 扩展性 | 新增 LLM 厂商仅增 Provider 类 + 配置枚举，不改生成主流程 |

## 9. 约束与假设

### 9.1 约束

- 开发期无微信 AppID、无备案域名  
- MVP 仅支持中文内容  
- 默认 LLM 为 DeepSeek，须支持切换其他已配置厂商  

### 9.2 假设

- 种子用户具备或可申请微信**服务号**  
- 用户接受小红书/抖音手动粘贴发布  
- 单租户用户数 < 20（MVP 不做复杂权限）
