> 关联文档：[v0.5-crm执行计划.md](./v0.5-crm执行计划.md)、[需求规格.md](./需求规格.md)、根目录 [README.md](../README.md)。

## 1. 部署架构

```
                    ┌─────────────┐     ┌─────────────┐
  浏览器 / 微信 H5  │  Web (Vue)  │     │  H5 (uni)   │
                    │  静态资源    │     │  静态资源    │
                    └──────┬──────┘     └──────┬──────┘
                           │  HTTPS             │
                           └────────┬───────────┘
                                    ▼
                           ┌────────────────┐
                           │ Nginx / 网关    │
                           │ 反代 /api       │
                           │ 反代 /storage   │
                           └────────┬───────┘
                                    ▼
                           ┌────────────────┐
                           │ FastAPI (8000)  │
                           │ uvicorn         │
                           └────────┬───────┘
                                    │
                    ┌───────────────┼───────────────┐
                    ▼               ▼               ▼
              PostgreSQL      STORAGE_DIR      外部 LLM API
              (生产必用)      (本地卷/OSS)     (DeepSeek 等)
```

| 组件 | 目录 | 构建产物 | 配置文件 |
|------|------|----------|----------|
| API | `apps/api/` | —（Python 进程） | `.env`（见 `.env.example`） |
| Web | `apps/web/` | `dist/` | `.env.production`（见 `.env.example`） |
| H5 | `apps/mp/` | `dist/build/h5/` | `.env.production`（见 `.env.example`） |

**Alembic 当前 head**：`033`（`033_persona_kb_seed.py`）。部署后须执行 `alembic upgrade head`。

> **v0.6 通用营销顾问**：迁移 `032_universal_advisor.py` 种子 marketing 顾问与通用场景模板；`033` 写入人格库 P-001～P-009。创作页统一使用 `marketing` 顾问，管理后台菜单为「营销顾问配置」。

## 4. 环境变量

配置读取逻辑见 `apps/api/app/config.py`（Pydantic Settings，自动加载 `apps/api/.env`）。

### 4.1 API（`apps/api/.env`）

复制模板：

```bash
cp apps/api/.env.example apps/api/.env
```

#### 必须修改（生产）

| 变量 | 说明 | 示例 |
|------|------|------|
| `DATABASE_URL` | PostgreSQL 连接串 | `postgresql+psycopg2://user:pass@host:5432/ai_marketing` |
| `JWT_SECRET` | JWT 签名密钥，**勿用默认值** | 随机长字符串 |
| `LLM_ENCRYPTION_KEY` | 租户 LLM Key 加密密钥，**≥32 字节** | 随机字符串 |
| `CORS_ORIGINS` | 允许跨域的前端域名，逗号分隔 | `https://admin.example.com,https://m.example.com` |

#### 建议配置

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DEEPSEEK_API_KEY` | 空 | 平台默认 LLM；也可登录后在「设置 → AI 模型」配置 |
| `LLM_PROVIDER` | `deepseek` | 也可配 OpenAI 兼容接口 |
| `LLM_BASE_URL` | `https://api.openai.com/v1` | OpenAI 兼容 Base URL |
| `LLM_API_KEY` | 空 | OpenAI 兼容 Key |
| `LLM_MODEL` | `gpt-4o-mini` | OpenAI 兼容模型名 |
| `LLM_TIMEOUT_SEC` | `60` | LLM 请求超时（秒） |
| `STORAGE_DIR` | `./storage` | **须挂持久卷**；存放发布内容、CRM 导入、导出 |
| `WECHAT_PUBLISHER` | `mock` | `mock` / `real`（真实微信需备案与服务号） |
| `PUBLISH_POLL_SEC` | `30` | 定时发布轮询间隔 |
| `JWT_EXPIRE_MINUTES` | `10080`（7 天） | Token 有效期 |

#### 短信（当前仅 Mock）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `SMS_PROVIDER` | `mock` | 上线前确认；暂无真实短信服务商集成 |
| `SMS_MOCK_CODE` | `1111` | Mock 验证码（忘记密码、注册等） |
| `SMS_CODE_EXPIRE_SEC` | `300` | 验证码有效期 |
| `SMS_SEND_INTERVAL_SEC` | `60` | 同号发码间隔 |

#### 生产 `.env` 示例

```ini
DATABASE_URL=postgresql+psycopg2://ai_marketing:强密码@127.0.0.1:5432/ai_marketing
JWT_SECRET=请替换为随机长字符串
LLM_ENCRYPTION_KEY=请替换为至少32字节的随机密钥
DEEPSEEK_API_KEY=sk-xxxxxxxx
CORS_ORIGINS=https://admin.yourdomain.com,https://m.yourdomain.com
STORAGE_DIR=/data/ai-marketing/storage
SMS_PROVIDER=mock
WECHAT_PUBLISHER=mock
```

> **勿在生产设置**（仅本地验收脚本使用）：`FORCE_FAKE_PLATFORM_LLM`、`VERIFY_LIVE_API`、`VERIFY_API_BASE`。

### 4.2 Web（`apps/web/.env.production`）

模板：`apps/web/.env.example`

| 变量 | 开发 | 生产 |

| `VITE_API_BASE_URL` | 空（走 Vite 代理） | **必填**，如 `https://api.yourdomain.com` |
| `VITE_API_PROXY_TARGET` | `http://127.0.0.1:8000` | 仅 dev |


### 4.3 H5（`apps/mp/.env.production`）

模板：`apps/mp/.env.example`，变量与 Web 相同，默认 dev 端口 `5174`。

```bash

npm run build:h5
```

> 若 `VITE_API_BASE_URL` 为空，前端请求同源 `/api`，需 Nginx 将 `/api`、`/storage` 反代到 API。

## 5. 部署步骤

### 5.1 数据库

```bash
# 创建库与用户（示例）
createdb ai_marketing
# 或使用 docker-compose 中的 postgres 服务

cd apps/api
alembic upgrade head    # 确认 current = 033 (head)
alembic current
```

### 5.2 启动 API

```bash
cd apps/api
# 确保 .env 已配置
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 2
```

验证：

```bash
curl http://127.0.0.1:8000/health
# 期望：{"status":"ok"}

curl http://127.0.0.1:8000/api/v1/crm/health
# 期望：{"status":"ok"}（需有效 Token 时见验收脚本）
```

### 5.3 静态资源

将构建产物部署到 Nginx / CDN：

| 端 | 构建命令 | 产物目录 |
|----|----------|----------|
| Web | `npm run build` | `apps/web/dist/` |
| H5 | `npm run build:h5` | `apps/mp/dist/build/h5/` |

### 5.4 存储目录

API 启动时会自动创建以下目录（基于 `STORAGE_DIR`）：

| 路径 | 用途 |
|------|------|
| `{STORAGE_DIR}/published/` | 已发布内容静态文件（挂载为 `/storage/published`） |
| `{STORAGE_DIR}/exports/` | 导出 zip（挂载为 `/storage/exports`） |
| `{STORAGE_DIR}/crm_imports/` | CRM 导入任务上传文件 |

**生产务必为 `STORAGE_DIR` 配置持久卷**，避免容器重建丢数据。

## 6. Nginx 参考配置

以下为「API 子域 + Web/H5 独立域名」方案；也可改为同源反代。

```nginx
# API
server {
    listen 443 ssl;
    server_name api.yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        # Agent SSE 需禁用缓冲
        proxy_buffering off;
        proxy_cache off;
        chunked_transfer_encoding on;
    }
}

# Web 管理端
server {
    listen 443 ssl;
    server_name admin.yourdomain.com;
    root /var/www/ai-marketing/web/dist;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }
}

# H5 移动端
server {
    listen 443 ssl;
    server_name m.yourdomain.com;
    root /var/www/ai-marketing/mp/h5;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

部署后确认 `CORS_ORIGINS` 包含 `https://admin.yourdomain.com` 与 `https://m.yourdomain.com`。

## 7. Docker Compose（可选）

根目录 `docker-compose.yml` 提供开发/演示用编排：

```bash
# 在项目根目录
export DEEPSEEK_API_KEY=sk-xxx   # 可选
docker compose up -d postgres api
```

| 服务 | 端口 | 说明 |
|------|------|------|
| `postgres` | 5432 | 用户/库：`ai_marketing` / `ai_marketing` |
| `redis` | 6379 | 当前 API **未使用**，可不启动 |
| `api` | 8000 | 启动时自动 `alembic upgrade head` |

**生产注意**：

1. 修改 `docker-compose.yml` 中 `JWT_SECRET`、`LLM_ENCRYPTION_KEY`、`CORS_ORIGINS`
2. 去掉 `volumes: ./apps/api:/app` 源码挂载
3. 修改 `apps/api/Dockerfile` 的 `CMD`：去掉 `--reload`，按需加 `--workers`
4. 为 `STORAGE_DIR` 挂命名卷

## 9. 回滚

| 场景 | 操作 |
|------|------|
| 仅前端问题 | 回退 `dist/` 静态资源到上一版本 |
| API 代码问题 | 回退 Python 包/镜像，**数据库一般不回滚** |
| 迁移有问题 | `alembic downgrade -1`（须确认降级脚本安全；生产慎用） |
| 配置错误 | 修正 `.env` 后重启 uvicorn |

建议：升级前备份 PostgreSQL 与 `STORAGE_DIR`。

## 11. 常见问题

| 现象 | 排查 |
|------|------|
| 前端 401 / CORS 报错 | 检查 `CORS_ORIGINS` 是否含前端域名；Token 是否过期 |
| 前端连不上 API | 确认 `VITE_API_BASE_URL` 或 Nginx `/api` 反代 |
| 迁移失败 | `alembic current` 与 `alembic heads`；数据库连接串是否正确 |
| 导入失败 | CSV 须含「公司名称」「联系人姓名」「手机」等必填列 |
| LLM 不可用 | 检查 `DEEPSEEK_API_KEY` 或租户设置页 Key；平台额度 |
| 忘记密码无效 | 当前 `SMS_PROVIDER=mock`，验证码固定 `1111` |
| 微信发布无效 | `WECHAT_PUBLISHER=mock` 为预期行为，真实接入需备案 |
