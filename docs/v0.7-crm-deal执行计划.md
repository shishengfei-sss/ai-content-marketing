### 0.3 业务闭环

```
线索 → 客户 → 商机 →─[最短]────────────→ 订单 → 收款计划 → 实际收款
            ↓                            ↑
          报价 ─[中 A]──────────────────┘
            ↓
          合同 ─[中 B / 最长]───────────┘

最长：商机 → 报价 → 合同 → 订单 → 收款
```

> 订单是「销售记录」的唯一锚点；收款挂在订单上。退货/开票/退款/红冲延后 v0.8。

### 0.4 关键决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 财务闭环 | 本次不做（回款做、开票/退款/红冲延后 v0.8） | 先打通销售链路 |
| 售后闭环 | 退货延后 v0.8（与退款/红冲一起做） | 退货依赖订单与已收回款联动 |
| 订单审批 | 无审批流 | `draft→confirmed` 销售自点 |
| 合同↔订单 | 一对多（合同可多次生成订单） | 支持续约/补单/分批结算 |
| 订单生成路径 | 4 路径并存（deal / quote / contract / quote→contract） | 最短到最长全覆盖 |
| 收款 | 挂订单；含回款计划与实际回款；确认归经理 | 防止销售自确认 |
| 看板拖拽 | vuedraggable | 体验最佳 |
| 报表图表 | ECharts（项目已安装） | 无新依赖 |

## 1. 数据模型

### 1.1 销售管道 `sales_pipelines`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id`, `tenant_id` | UUID | — |
| `name` | str | 管道名 |
| `is_default` | bool | 租户内仅 1 条默认 |
| `is_active` | bool | — |
| `created_at`, `updated_at` | datetime | — |

### 1.2 管道阶段 `sales_pipeline_stages`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id`, `pipeline_id`, `tenant_id` | UUID | — |
| `name` | str | 阶段名 |
| `sort_order` | int | 顺序 |
| `probability` | int | 默认成交概率 0~100 |
| `is_won_stage` | bool | 赢单终态 |
| `is_lost_stage` | bool | 输单终态 |
| `is_active` | bool | — |

**种子：** 默认管道 + 6 阶段（qualifying 20% / presenting 40% / quoted 55% / negotiating 70% / closed_won 100% / closed_lost 0%）。

### 1.3 商机 `deals`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id`, `tenant_id` | UUID | — |
| `title` | str | 商机名（主属性） |
| `customer_id` | UUID FK | 必填 |
| `contact_id` | UUID FK nullable | 主要联系人 |
| `pipeline_id`, `stage_id` | UUID FK | 当前管道与阶段 |
| `amount` | numeric(14,2) | 商机金额（元） |
| `expected_close_date` | date | 预计成交日 |
| `probability` | int | 当前概率 |
| `status` | str | `open` / `won` / `lost` / `abandoned` |
| `loss_reason` | str nullable | 输单原因 |
| `source` | str nullable | 来源 |
| `campaign_id` | UUID nullable | 营销活动归因 |
| `owner_user_id`, `territory_id` | UUID | 参与 scope |
| `converted_from_lead_id` | UUID nullable | 由线索转化时记录 |
| `converted_order_id` | UUID nullable | 转出订单（可多次，非唯一） |
| `extra_data` | JSON | 自定义字段 |
| `created_by_user_id`, `created_at`, `updated_at`, `deleted_at` | — | — |

### 1.4 商机阶段流转日志 `deal_stage_logs`

`deal_id`, `from_stage_id`, `to_stage_id`, `changed_by_user_id`, `changed_at`, `note`

### 1.5 产品 `products`

`code`/`name`/`unit`/`list_price`/`cost_price?`/`is_active`/`description`/`extra_data`；`code` 租户内唯一。

### 1.6 报价 `quotes` + `quote_lines`

**`quotes`**：`quote_number`/`deal_id?`/`customer_id`/`contact_id?`/`subject`/`discount_rate?`/`total_amount`/`status`(draft/sent/accepted/rejected/expired/ordered)/`valid_until`/`owner_user_id`/`converted_order_id?`/`extra_data`

**`quote_lines`**：`product_id?`/`name`/`unit`/`quantity`/`unit_price`/`discount_rate`/`line_total`/`sort_order`/`remark`

服务端计算：`line_total = quantity × unit_price × (1 - discount_rate/100)`；`total = Σ(line_total) × (1 - discount_rate/100)`。

### 1.7 合同 `contracts`

`contract_number`/`deal_id?`/`customer_id`/`quote_id?`/`title`/`contract_type`(new/renewal/addon)/`amount`/`signed_amount?`/`start_date`/`end_date`/`status`(draft/sent/signed/executing/expired/terminated)/`signed_at?`/`owner_user_id`/`file_url?`/`extra_data`

合同可多次生成订单（`orders.contract_id` 反向引用，一对多）。

### 1.8 订单 `orders` + `order_lines`

**`orders`**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id`, `tenant_id` | UUID | — |
| `order_number` | str | 自动 `O{yyyymmdd}NNN` |
| `title` | str | 订单主题 |
| `customer_id` | UUID FK | 必填 |
| `contact_id` | UUID FK nullable | — |
| `deal_id` | UUID FK nullable | 至少填一个源单 |
| `quote_id` | UUID FK nullable | 由报价转来 |
| `contract_id` | UUID FK nullable | 由合同转来 |
| `source` | str | `deal` / `quote` / `contract` |
| `order_date` | date | 下单日 |
| `amount` | numeric(14,2) | 服务端汇总 |
| `status` | str | `draft` / `confirmed` / `executing` / `completed` / `cancelled` |
| `owner_user_id`, `territory_id?` | UUID | 参与 scope |
| `extra_data` | JSON | — |
| `created_by_user_id`, `created_at`, `updated_at`, `deleted_at` | — | — |

**`order_lines`**：`product_id?`/`name`/`unit`/`quantity`/`unit_price`/`discount_rate`/`line_total`/`sort_order`/`remark`

**状态机（无审批）：**

```
draft → confirmed → executing → completed
                  ↘ cancelled（已有 confirmed 回款 → 409）
```

### 1.10 业务规则

| ID | 规则 |
|----|------|
| BR-DEAL-01 | 商机默认 `owner=当前用户`；默认管道 `is_default=true`；默认第一阶段 |
| BR-DEAL-02 | 关闭赢单必填 `amount` + `closed_at`；输单必填 `loss_reason` |
| BR-DEAL-03 | 阶段推进写 `deal_stage_logs`；`probability` 默认取新阶段值 |
| BR-QUOTE-01 | 报价金额服务端计算，前端传值忽略 |
| BR-QUOTE-02 | 报价转订单：复制 `quote_lines`→`order_lines`，`order.amount=quote.total_amount`，`quote.status=ordered` |
| BR-CONTRACT-01 | 合同可多次生成订单 |
| BR-ORDER-01 | 订单必填 `customer_id` + 至少一个源单 |
| BR-ORDER-02 | 取消订单：已有 confirmed 回款 → 409；需先全部 reversed |
| BR-PAY-01 | 回款必须挂 `confirmed/executing/completed` 状态订单 |
| BR-PAY-02 | 累计已收（含 confirmed，扣 reversed）≤ `order.amount`；超额 → 409 |
| BR-PAY-03 | `pending→confirmed` 需 `crm.payment.confirm`（销售不可自确认） |
| BR-PAY-04 | `confirmed→reversed` 冲销；冲销后该笔不计入已收 |
| BR-PAY-05 | 回款 scope 继承订单可见性 |

### 1.11 entity_field_definitions 扩展

新增 `entity_type`：`deal` / `quote` / `contract` / `order` / `payment` / `product`，复用 §3.15 自定义字段、列偏好、保存视图。

## 6. v0.8 财务闭环与售后（预告）

为避免遗留设计冲突，本次 `orders` 与 `payments` 已预留对接位：

- 后续 `return_orders` 挂 `order_id`，可关联 `refunds` 形成「退+退款」联动
- 后续 `invoices` 挂 `order_id`；`credit_notes` 关联 `invoices`
- 应收台账 = `orders.amount - payments.paid` 衍生
- 客户对账单 = `orders + payments + returns + refunds + invoices + credit_notes` 汇总
- 坏账、预收款/定金、回款认领也基于本次 `payments` 扩展
